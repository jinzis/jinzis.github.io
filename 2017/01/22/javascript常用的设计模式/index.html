javascript常用的设计模式

发表于 2017-01-22

单例模式

把描述同一事物（同一对象）的属性和方法放在一个内存空间下（命名空间），起到分组的作用。

单例模式是工作中经常使用的一种模式，单例模式能够避免多人开发，全局变量污染等问题。
单例模式也有它的缺点，不能实现批量生产。

工厂模式

把实现同一件事情的相同的代码放到同一个函数中，以后只需执行这个函数即可

工厂模式虽然解决了创建多个相似对象的问题，却没有解决对象识别的问题。

构造函数模式

为了更好的理解构造函数模式，首先要理解类和实例的概念。
类：具有相同属性的一类事物的集合。类都是函数数据类型的，有对象类Object，数组类Array等等。
实例：是类的细分，都是对象数据类型的。

构造函数在执行的时候需要new一个函数来执行，构造函数不需要手动返回一个对象，浏览器会自动返回一个对象。
构造函数的属性名都绑定在当前类的实例上。
构造函数虽然解决了实例识别的问题，但构造函数也有它的缺点，每个方法都要在实例上重新创建一遍。

原型模式

我们创建的每一个函数都有一个prototype（原型）的属性，这个属性指向一个对象。这个对象里面存储了公有的属性和方法。
在prototype上浏览器默认给它一个constructor（构造函数）的属性，它的值是类本身。
每一个对象数据类型（普通对象，实例，原型）都有一个自带的属性__proto__，它的值是当前实例所属类的原型。
原型模式虽然解决了属性和方法公有的问题，但它省略了为构造函数传递初始化参数的环节，导致所有实例默认情况下都将取得相同的属性值。

构造函数+原型的混合模式

通过对象名.属性名的方式获取属性值的时候，先在自己的私有属性上找，
私有的有就用私有的，若私有的没有就到它的类的原型上找公有的，若公有的没有就到Object的原型上找。
